  <behavior>
    `confirm_prompt(message: str, *, console: Console | None = None) -> bool`:
    - Calls `input(f"{message} [y/n] ")`.
    - Returns `True` for input `"y"`, `"Y"` (and `" y "` after `.strip().lower()`).
    - Returns `False` for input `"n"`, `"N"`.
    - On `""` or any other character: re-prompts (loop until valid input).
    - On `KeyboardInterrupt` from `input()`: prints a blank line to `console`, then `Text("Operation aborted. No changes were made.")`, returns `False`.

    `text_prompt(label: str, *, console: Console | None = None) -> str` (rule 6.16):
    - Calls `input(f"Enter {label}: ")`, returns the stripped result. Re-prompts on empty input.

    `number_prompt(label: str, unit: str, *, console: Console | None = None) -> float` (rule 6.16):
    - Calls `input(f"{label} in {unit}: ")`; converts via `float(raw)`. On `ValueError`, prints `Error: Expected a number.` via `print_error` (if available) OR `console.print(...)` and re-prompts.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `input()` → confirm_prompt | Untrusted user keystrokes; KeyboardInterrupt always possible |
| `input()` → number_prompt | Untrusted strings; must validate as float, no eval |
| caller `columns`/`rows` → Rich Table | Caller-supplied strings; Rich Table renders cells safely (markup is opt-in) |

## STRIDE Threat Register (ASVS L1)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | `number_prompt(label, unit)` parsing | mitigate | Uses `float(raw)` directly inside `try/except ValueError`; no `eval`/`exec`. Invalid input loops to re-prompt. ASVS V5 input validation. |
| T-05-08 | DoS | `confirm_prompt` infinite loop | accept | Loop exits only on `y/Y/n/N` or `KeyboardInterrupt`. v1 is a single-user CLI; the user controls their own keyboard. No remote attacker. |
| T-05-09 | Spoofing | `confirm_prompt` accepts default Enter as Yes | mitigate | Rule 6.15 says "No default. Pressing Enter without input re-prompts." Implementation rejects `""` (empty after strip) and loops. Test `test_confirm_reprompts_on_invalid` covers the empty case. |
| T-05-10 | Tampering | `render_table` cell value with Rich markup | accept | `Table.add_row(*row)` passes strings as-is; Rich renders markup if present. Cells in v1 come from internal formatters (`format_price`, `format_pnl`, etc.) which produce safe strings. External strings (symbols, error messages) go through `print_message`, not tables. Document the assumption; Phase 13 hardening adds explicit `markup=False` if external input is ever rendered. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed. |
</threat_model>
