  <action>
    Create src/bensdorp1/commands/_scan_engine.py. This is a new file. The module must NOT import from bensdorp1._app or typer — it is the engine layer only, no CLI concerns.

    Module-level constants: SEPARATOR = "=" * 64 (matches init.py and spec §7.2), PRICE_FETCH_WINDOW_DAYS = 10.

    Public entry point: run_scan(engine: Engine, *, force: bool = False, console: Console | None = None) -> str. Inside: con = console or Console() for progress, capture = Console(record=True, width=120) for rendered output. Returns capture.export_text().

    The 7 internal helpers in call order from run_scan:

    _run_preflight(engine, con, today, last_scan_date) -> tuple[dict[str, str], list[int], list[str]]:
      Returns (constituents, missed_trading_days_as_date_list, notes_list). Steps:
      1. Check constituents freshness: query constituents_cache MAX(fetched_at); if older than 7 days, call get_constituents(engine). Otherwise, use SELECT from constituents_cache.
      2. Call check_price_coverage(engine): if covered/total < 0.95, raise RuntimeError with message "Price coverage below 95%: {covered}/{total}. Run `bensdorp1 init` or wait for next scan."
      3. Detect catch-up: query MAX(scan_date) from scans where scan_date < today_midnight_utc. If last scan was 2+ trading days ago, call get_trading_days(last_scan_date + timedelta(1), yesterday) to get missed days list. Add note per D-10 if catch-up happened.

    _get_available_cash(engine) -> float:
      SELECT config.value WHERE key='available_cash'; return float(row.value) or 0.0.

    _load_price_dfs(engine, symbols, rows_needed=221) -> dict[str, pd.DataFrame]:
      For each symbol, SELECT trade_date, close, volume FROM price_daily WHERE symbol=? ORDER BY trade_date ASC; take df.tail(rows_needed). Build dict. Return only symbols that have data. Columns must be named "close" and "volume" (lowercase) — Pitfall 1. Note: trade_date is DateTime(timezone=True) — use .date() for comparisons if needed.

    _fetch_data(engine, con, constituents, today) -> None:
      Calls update_price_data(engine, all_symbols, start=date.today()-timedelta(days=PRICE_FETCH_WINDOW_DAYS), end=date.today()) where all_symbols = list(constituents.keys()) + ["^GSPC"]. Wraps in feedback.multi_step(2, console=con) step [1/2] "Fetching latest market data" with total=len(all_symbols). After the step block exits, print "      Constituents fetched: {n}/{total}" to con. Step [2/2] "Computing signals" is in _run_screening. Per D-13: call update_price_data once for all symbols (bulk), then advance track for each symbol after the bulk call. Per Pitfall 6: always pass both start and end.

    _update_position_stops(engine, open_positions, missed_days, today, price_dfs, triggered_position_ids) -> None:
      Iterates over missed_days then today. For each day and each open position not in triggered_position_ids: get close from price_dfs[symbol] for that date (match via trade_date.date()). If no price data for that day, continue. Compute new_hc = update_highest_close(pos.highest_close, close), new_ts = compute_trailing_stop(new_hc), eff = compute_effective_stop(pos.initial_stop, new_ts). If is_exit_triggered(close, eff): add to triggered_position_ids set; do NOT update positions row (freeze). Else: UPDATE positions SET highest_close=new_hc, trailing_stop=new_ts WHERE id=pos.id using SQLAlchemy update() with bound params.

    _detect_exit_triggers(engine, open_positions, triggered_position_ids, scan_id, today, price_dfs) -> list[dict]:
      For positions where pos.id in triggered_position_ids and no existing row in scan_exit_triggers for pos.id: determine reason — "Trailing stop" if pos.trailing_stop >= pos.initial_stop else "Initial stop". Determine effective_stop = compute_effective_stop(pos.initial_stop, pos.trailing_stop). Determine triggered_date: the earliest date in missed_days (or today) when is_exit_triggered first returned True — this is the actual day the stop was hit. INSERT into scan_exit_triggers (scan_id=scan_id, position_id=pos.id, reason=reason, effective_stop=effective_stop, triggered_date=triggered_date_utc). Also return existing scan_exit_triggers rows for open positions that already have a trigger (pending triggers from prior scans) for display.

    _run_screening(engine, con, spinner, constituents, today, cash) -> tuple[bool, float, float, list[Candidate]]:
      Load price_dfs for all constituents + ^GSPC. Pass spx DataFrame close Series to regime_filter. If regime_active: run liquidity_filter -> momentum_filter -> rank_candidates(filtered_dfs, cash). Returns (regime_active, spx_close, spx_sma_200, candidates). SPX close = spx_df["close"].iloc[-1]; SPX SMA 200 = spx_df["close"].tail(200).mean(). Wrap filter calls in spinner.tick(). Per Pitfall 1: always load from price_daily (DB), not yfinance. Per Pitfall 8: _update_position_stops already computed triggered_position_ids before this function runs.

    _render_output(capture, today, regime_active, spx_close, spx_sma_200, today_triggers, pending_triggers, candidates, cash, catch_up_notes) -> None:
      Renders all output sections to capture console. Section order per spec §7.2:
      1. Header: SEPARATOR, "Scan for {format_date(today)}", SEPARATOR
      2. "Market regime" + "-------------"; render_kv_block with "S&P 500 close": f"{spx_close:,.2f}" (no $), "S&P 500 SMA 200": f"{spx_sma_200:,.2f}" (no $), "Regime": "Bull market (close above SMA 200)" or "Bear market (close below SMA 200)"
      3. If today_triggers: "Exit triggers" + "-------------"; render_table with columns [Symbol/left, Reason/left, Close/right, Effective stop/right, Days held/right]; footer line about next market open + bensdorp1 sell instruction
      4. If pending_triggers (from prior scans, not today): "Pending exit triggers from previous scans" section with render_table columns [Symbol/left, Triggered on/left, Reason/left, Original stop/right]; "Run `bensdorp1 sell SYMBOL PRICE` to confirm the exit." line
      5. If regime_active: "Buy candidates (top 10)" table with columns [Rank/right, Symbol/left, Close/right, ROC 200d/right, Volume (avg 20d)/right]; "Buy candidates affordable (cash: {format_price(cash)})" table with columns [Rank/right, Symbol/left, Close/right, ROC 200d/right, Shares to buy/right] — only rows where candidate.position_size > 0; if all position_size == 0, show note per D-22
      6. "System notes" + "------------"; print each note from catch_up_notes; print "No catch-up actions needed." if none; print "Constituents list verified successfully."
      All strings from DB (symbol, company names) must pass through Text(str) — never raw string to capture.print. SPX close/SMA uses f"{value:,.2f}" not format_price(). Stock prices use format_price(). Percentages use format_pct(). Volumes use format_volume(). Days held uses format_days().

    _persist_scan(engine, today, regime_active, raw_output, candidates, new_trigger_count, constituents, spx_close, spx_sma_200, freshness_days) -> int:
      Uses sqlite_insert(scans).values(...).on_conflict_do_update(index_elements=["scan_date"], set_={...}) per D-02. INSERT scan_candidates rows for each candidate (delete existing for same scan_id first if force). Call log_event with AuditEventType.SCAN_PERFORMED and full payload per spec §9.2. Call create_backup(engine, DATA_DIR / "backups"). Return inserted scan_id.

    run_scan coordinates all helpers in order: _run_preflight -> _fetch_data (inside multi_step step 1) -> _update_position_stops + _detect_exit_triggers -> _run_screening (inside multi_step step 2) -> _render_output to capture -> _persist_scan -> return capture.export_text(). All within a try/except that converts RuntimeError to a plain error message string returned to scan.py for printing.

    Type annotations: all functions fully typed. Use Any only where SQLAlchemy Row types are needed; prefer explicit field access. No bare except. mypy strict must pass.
