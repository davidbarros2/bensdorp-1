    Create .github/workflows/ci.yml (create .github/workflows/ directory as needed):

    Name: CI
    on: push (branches: ["main"]) and pull_request (all branches)

    jobs.test:
      name: test (${{ matrix.os }})
      runs-on: ${{ matrix.os }}
      strategy: fail-fast: false, matrix: os: [ubuntu-latest, windows-latest]
      steps:
        1. uses: actions/checkout@v4
        2. Install uv: uses: astral-sh/setup-uv@v6 with: python-version: "3.11", enable-cache: true, cache-dependency-glob: "uv.lock"
        3. Install project: run: uv sync --locked --dev
        4. Run pytest: run: uv run pytest tests/ -v
        5. Run ruff check: run: uv run ruff check .
        6. Run ruff format check: run: uv run ruff format --check .
        7. Run mypy: run: uv run mypy src/

    Key constraints:
    - fail-fast: false so both OS jobs complete even if one fails
    - uv sync --locked --dev (not just uv sync) — --locked enforces uv.lock is current; --dev installs the dev dependency group
    - mypy runs against src/ (not . or repo root) — avoids false positives in tests/
    - astral-sh/setup-uv@v6 (not v5 or v4) — current stable version per RESEARCH.md Pattern 7
    - No shell: override is needed — uv run handles cross-platform activation on both ubuntu and windows

    Create .github/workflows/close-pr.yml:

    Name: Close Pull Request
    on: pull_request_target: types: [opened, reopened]
    (CRITICAL: pull_request_target not pull_request — forks get write tokens only with pull_request_target)

    jobs.close:
      runs-on: ubuntu-latest
      steps:
        - uses: superbrothers/close-pull-request@v3
          with:
            comment: |
              Thank you for your interest in bensdorp1.

              This is a personal tool and does not accept external contributions. Pull requests are closed automatically.

              If you have found a bug or have a suggestion, please contact the maintainer directly.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| fork PR → close-pr.yml | External users can open PRs; workflow runs with write token via pull_request_target |
| CI runner → uv.lock | Lockfile controls exact package versions installed in CI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | close-pr.yml using pull_request_target | mitigate | Use pull_request_target (not pull_request) — runs from base branch with write token; fork PRs cannot exfiltrate repo secrets because no secrets are passed to this workflow |
| T-04-02 | Elevation of Privilege | GitHub Actions GITHUB_TOKEN | mitigate | close-pr.yml only needs write permission to close PRs and post comments; no secrets beyond GITHUB_TOKEN (implicit); branch protection prevents unauthorized merges to main |
| T-04-03 | Tampering | ci.yml uv sync --locked | mitigate | --locked flag enforces uv.lock hash-pinned versions; a modified pyproject.toml without a re-locked uv.lock fails CI loudly |
| T-04-04 | Tampering | Branch protection disabled | mitigate | Manual step in checkpoint task requires user to enable branch protection requiring CI pass; enforces that no code lands in main without green CI |
| T-04-05 | Spoofing | Malicious fork PR | mitigate | pull_request_target + close-pr.yml auto-closes within seconds; branch protection prevents merge even if CI is bypassed |
| T-04-SC | Tampering | GitHub Actions third-party actions | mitigate | actions/checkout@v4, astral-sh/setup-uv@v6, superbrothers/close-pull-request@v3 — all pinned to major version tags; slopcheck audit covers Python packages; GitHub Actions actions are not in scope of slopcheck but are established actions with public audit trails |
</threat_model>
