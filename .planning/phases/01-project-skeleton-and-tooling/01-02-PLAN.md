| Boundary | Description |
|----------|-------------|
| user input → help command | The `command` argument to `bensdorp1 help <cmd>` is user-supplied text; it is used as a dict key lookup, not passed to subprocess in Approach B |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | help command Approach B (ctx.find_root().command.commands.get) | mitigate | Uses dict key lookup — no subprocess invocation; user input cannot escape to shell; # type: ignore[attr-defined] is necessary but the attribute is a read-only dict lookup, not exec |
| T-02-02 | Tampering | help command Approach A fallback (subprocess) | accept | Approach B is used (no subprocess); if team ever switches to Approach A, must add explicit command name allowlist validation before passing to subprocess |
| T-02-03 | Spoofing | Stub commands printing "Not yet implemented." | accept | Stubs are expected placeholders — exit code 0 is intentional per D-05; no security boundary at this layer |
