<objective>
Implement `src/bensdorp1/data/constituents.py` — the S&P 500 constituent fetcher, cross-checker, and 7-day cache manager. This module is one of three I/O-bearing data layer modules; it covers Wikipedia primary scraping, Slickcharts cross-check, the 0-3/4-10/11+ discrepancy gate (DATA-02), and the 7-day cache TTL (DATA-05). On network failure with a populated cache, it falls back silently and emits a DATA_FETCH_FAILED audit event — never hard-fails (D-03).

Purpose: Phase 7 (scan) calls `get_constituents()` once per scan; Phase 10 (refresh command) calls `refresh_constituents()` to force a re-fetch. This plan delivers both. The discrepancy disposition is exposed to callers via the CONSTITUENTS_DISCREPANCY audit event payload (`severity` field set to "warn" or "abort") — the caller decides whether to suppress buy candidates. The data layer never decides scan flow, only signals via audit events.

Output: 1 new module (`src/bensdorp1/data/constituents.py`) and 1 rewritten test file (`tests/test_data_constituents.py` — overwrites the Plan 01 scaffold). DATA-01, DATA-02, and DATA-05 are fully implemented and tested.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Wikipedia HTTP → process | Wikipedia HTML is untrusted: anyone can edit the S&P 500 page. Symbols may be malformed, duplicated, or contain injection payloads. |
| Slickcharts HTTP → process | Slickcharts HTML is untrusted and rate-limited; may serve a Cloudflare challenge page on bot detection. |
| process → SQLite | All inserts/deletes on `constituents_cache` cross this boundary; SQL injection via ticker symbols is a primary concern. |
| process → audit_log | Audit payloads include scraped strings (wikipedia_only, slickcharts_only ticker lists); these reach audit_log.payload as JSON. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-CONST-01 | Tampering | SQL injection via ticker symbol from scraped HTML | mitigate | All inserts use SQLAlchemy Core `insert(constituents_cache).values(rows)` with dict-based parameters — never f-strings. Plus pre-insert validation via `_TICKER_RE = re.compile(r"^[A-Z.\-\^]{1,10}$")` rejects any value containing `;`, `'`, spaces, lowercase, or length>10. Verified by `test_fetch_wikipedia_drops_malformed_tickers`. |
| T-03-CONST-02 | Tampering | XSS via scraped company_name field | mitigate | BeautifulSoup `get_text(strip=True)` strips all HTML tags; only plain text reaches `constituents_cache.company_name`. The data layer does not render HTML — Phase 5 (UI) handles display escaping. |
