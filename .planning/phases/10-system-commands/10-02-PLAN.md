<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External HTTP (Wikipedia, Slickcharts) → constituent symbols | Untrusted upstream input; refresh_constituents already validates each symbol via `_validate_ticker` regex before persisting |
| DB row values (constituents_cache.symbol) → console output via render_table | Symbol strings rendered in table; render_table uses Text wrapping (tables.py line 40) |
| User CLI invocation → command entry | Single-user CLI; no privilege boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Tampering | Rich markup injection via constituent symbols containing `[` in the Added/Removed table | mitigate | render_table wraps every cell via `Text(cell)` (tables.py line 40), which disables markup interpretation. Symbol validator in constituents.py `_validate_ticker` regex `^[A-Z.\-\^]{1,10}$` already excludes `[` and `]`. Double-layer defence sufficient. |
| T-10-02-02 | Tampering | Stale or poisoned Wikipedia / Slickcharts response causes wrong tickers persisted | mitigate | refresh_constituents already classifies discrepancy via DATA-02 (0-3 silent, 4-10 warn, 11+ abort) and logs CONSTITUENTS_DISCREPANCY. Phase 10 inherits this mitigation; no new mitigation needed. |
| T-10-02-03 | Denial of Service | Network timeout in refresh_constituents hangs the CLI | accept | refresh_constituents uses `timeout=30.0` on httpx.Client (constituents.py line 158). 30 s ceiling per fetch is acceptable for an on-demand command. Spinner provides user feedback after 1 s. |
| T-10-02-04 | Information Disclosure | DATA_FETCH_FAILED audit events expose error details | accept | Single-user local CLI; audit log is owned by the user; revealing fetch errors is desirable for diagnosis. |
</threat_model>
