## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → backups_dir read | Directory listing pulled from local disk; mtimes and filenames not under attacker control in single-user CLI but malformed filenames could break formatting |
| SQLite file → PRAGMA integrity_check output | DB file content read into status text output |
| DB row values (e.g. constituents_cache.symbol, scans.scan_date) → console output | Untrusted-looking values rendered via render_kv_block |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | Tampering | Rich markup injection via DB values in status output (e.g. constituent symbol containing `[` chars) | mitigate | render_kv_block already calls `console.print(..., markup=False, highlight=False)` (ui/styles.py line 166). All section headers and separators in status.py MUST also use `Text(...)` + `markup=False, highlight=False`. No bare `console.print(string_literal)` calls — verified in acceptance criteria. |
| T-10-01-02 | Denial of Service | Empty/missing backups_dir → `max([])` ValueError crashes status | mitigate | Explicit `backups_dir.exists()` guard before glob and explicit empty-list check before indexing `db_files[0]` (Pitfall 5). Acceptance criterion checks for guard. |
| T-10-01-03 | Denial of Service | Multi-row PRAGMA integrity_check result silently misclassified as 'ok' by `.scalar()` | mitigate | Use `.fetchall()` and require `len(rows) == 1 and rows[0][0] == "ok"` (Pitfall 4). Acceptance criterion greps for `fetchall` in status.py. |
| T-10-01-04 | Information Disclosure | Backup file paths (absolute) printed in status output reveal home directory | accept | Single-user local CLI; user owns their own filesystem; no remote disclosure vector. Path is required for user to find their backups. |
